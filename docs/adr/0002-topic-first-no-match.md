# Topic-first 產品，不做 swipe / match

產品用公開的 **topic feed**（Channels + Posts + Comments）取代 Tinder 式 swipe 配對——使用者透過公開對話互相發現，再選擇是否 DM。**沒有** swipe、沒有 match、沒有推薦卡片。

這是產品的核心差異化：模型更簡單、心理門檻更低（「低配對焦慮」）、portfolio 實作更快。對應 deck slide 11——「輕鬆的交友體驗：將探索取代配對頁設定為首頁，除去『標籤化、急促交友』，成為『低配對焦慮』的交友體驗。」

## 評估過的替代方案

- **Buddy-first（Tinder + topic 過濾）**——保留 swipe / match 比喻，加上 topic 風味的篩選。否決：跟既有 app 的差異化太薄、實作成本（match engine、swipe UX）撐不起 portfolio 投入。
- **Hybrid（topic feed + free DM）**——表面上跟最終設計接近，但仍保留「夥伴」這個帶 match 語意的詞。否決：deck 上「夥伴」一詞已過載，留著會讓資料模型曖昧不清。
