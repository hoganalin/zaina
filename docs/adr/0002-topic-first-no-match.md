# Topic-first product, no swipe / no match

The product replaces Tinder-style swipe matching with a public **topic feed** (Channels + Posts + Comments) where users discover each other through public conversation, then optionally DM. There is no swipe, no match, no recommendation card.

This is the product's core differentiation: a simpler model, lower psychological barrier ("低配對焦慮"), and faster portfolio implementation. Confirmed against the team's pitch deck slide 11 — "輕鬆的交友體驗：將探索取代配對頁設定為首頁，除去『標籤化、急促交友』，成為『低配對焦慮』的交友體驗."

## Considered alternatives

- **Buddy-first (Tinder + topic filter)** — kept the swipe / match metaphor but added topic-flavoured filtering. Rejected because the differentiation against existing apps is too thin and the implementation cost (matching engine, swipe UX) doesn't earn its keep in a portfolio.
- **Hybrid (topic feed + free DM)** — similar surface area to the chosen design, but kept the term "夥伴" with match semantics. Rejected because the term was overloaded across the deck and would have left the data model fuzzy.
